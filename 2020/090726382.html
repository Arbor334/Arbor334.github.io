帮助手册

命令模式中：

输入字符方式 产生效果
u 相当于windows中的 撤销
ctrl+r 相当于windows中的前进
i 从光标当前的位置插入
I 从光标的开头插入
a 当前光标的下一个字符插入
A 光标本行的结束插入
o 在下一行产生一个空白行，并在这个空白行插入
O 在下一行产生一个空白行，但是实在本行插入

一些其他的用法

输入 效果
：4 切换到第4行
：set nu 在行首出现序号
：set nonu 行首序号消失
：s/xxx/yyy/ 将当前行的第一个xxx替换为yyy
：s/xxx/yyy/g 将当前行的所有的xxx都替换为yyy
:3,13s/xxx/yyy/g 从第3行到第13行的xxx都替换为yyy
：.,$s/xx/yyg 从”.”(当前行)到”$”(最后一行)xx都替换为yy
：%s/xx/yy/g 整篇文章xx都替换yy %就等于(1,$)
：%s/xx//g 删除全文的xx
：.,$s/^/#/g ^（开头）#(注释) 全都都加上注释
：.,$s/^#//g 将开头是#的删除
yy 复制当前行
4yy 当前行向下数3行 四行都复制
p 粘贴复制的数据
dd 剪切当前行
4dd 当前行下数3行 四行都剪切
r x 按一下r键 在输入想要替换的字符 就可以替换了
x 在命令行 按x就删除了字符
1,. 删除从第一行到当前行的所有数据 .代表的是当前行
输入 效果
/ xxx 查找xxx（n从上往下查找，N从下往上查找）
15gg 在命令模式输入15 再按两下g 就切换到了15行·

shell是一个翻译器，把我们所敲的命令 翻译成cpu可以看懂的 语言

将环境变量变为本地变量 declare +x 变量名

set 3 2 3 4 5 6 7 8 9 a b c

echo $10

输出的却是30 ？为什么 呢？

因为 解析的时候先解析$1 是3 后面的0解析不出来 直接付过去

为了避免产生歧义 echo ${10} 用大括号括起来

shell编写

vim xx.sh //shell文件的后缀是.sh

#!/bin/bash //在开头要写的

chmod +x xx.sh 使其具有执行功能

运行 ./xx.sh

注意，一定要写成 ./test.sh，而不是 test.sh，运行其它二进制的程序也一样，直接写 test.sh，linux 系统会去 PATH 里寻找有没有叫 test.sh 的，而只有 /bin, /sbin, /usr/bin，/usr/sbin 等在 PATH 里，你的当前目录通常不在 PATH 里，所以写成 test.sh 是会找不到命令的，要用 ./test.sh 告诉系统说，就在当前目录找

PS1

更改PS1的格式

PS1=’[\u@\h \w]’

\d ：代表日期，格式为weekday month date，例如：”Mon Aug 1”

　　\H ：完整的主机名称。例如：我的机器名称为：arbor1.linux，则这个名称就是arbor1.linux

　　\h ：仅取主机的第一个名字，如上例，则为arbo1，.linux则被省略

　　\t ：显示时间为24小时格式，如：HH：MM：SS

　　\T ：显示时间为12小时格式

　　\A ：显示时间为24小时格式：HH：MM

　　\u ：当前用户的账号名称

　　\v ：BASH的版本信息

　　\w ：完整的工作目录名称。家目录会以 ~代替

　　\W ：利用basename取得工作目录名称，所以只会列出最后一个目录

　　# ：下达的第几个命令

　　$ ：提示字符，如果是root时，提示符为：# ，普通用户则为：$
