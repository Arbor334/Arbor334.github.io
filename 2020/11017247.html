c++内存四区

内存分区模型

代码区：存放函数的二进制代码，有操作系统进行管理

全局区：存放全局变量和静态常量以及常量

栈区：由编译器自动分配释放，存放函数的参数值，局部变量等

堆区：有程序员分配和释放，若程序员不释放，程序结束时由操作系统回收

程序执行前

在程序编译后，生成了exe可执行文件，未执行该程序前分为两个区域。

代码区和全局区

一、代码区

存放cpu执行的指令 (01代码)

代码区是共享的，共享的目的是对于频繁被执行的程序，只需要在内存中有一份代码即可

代码区是只读的，使其只读的原因是方式程序以外的修改它的指令

二、全局区

全局变量和静态常量（static）存放在此

全局区还包含了常量区，字符串常量和其他常量在存放在此。

该区域的数据在程序结束够由`操作系统释放

  • 在全局区中的数据

    • 全局变量

    • 字符串常量

    • const修饰的全局常量

    • static表示的静态常量

  • 不在全局区中的数据

    • 局部变量
    • const修饰的局部常量
#include<algorithm>
#include<iostream>
using namespace std;
int g_a = 10;//全局变量
const int c_g_a = 10;//const修饰的全局常量
int main() {
    static int a = 10;//static 表示的静态常量
    cout << "全局变量：" << (int)&g_a << endl;
    cout << "const 修饰的全局常量的地址为：" << (int)&c_g_a << endl;
    cout << "static 表示的静态常量"<<(int)&a << endl;
    cout << "字符串常量的地址:" << (int)&"hello world" << endl;
}

三、栈区

由编译器自动分配释放，存放函数的参数值，局部变量等

注意事项，不要反悔局部变量的地址，栈区开辟的数据有编译器自动释放

四、堆区

有程序员分配释放，若程序员不释放，程序结束时有操作系统回收

在c++中主要利用new 在堆区开辟内存

#include<algorithm>
#include<iostream>
using namespace std;
int* func() {
     //利用new关键字 可已经数据开辟到堆区
      // 指针 本职业是局部变量，放在栈上，指针保存的数据是放在堆区的
  int* p = new int(10); //创建好以后，返回的是堆区的地址

    return p;

}
int main() {
    int *p = func();
    cout << *p << endl;
}
