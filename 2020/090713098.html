linux 的账号管理和ACL权限设置[1]

linux账号与用户组

用户标识符UID 和GID

每个用户都有自己的UID

​ root 用户的UID 为0

​ 系统用户的UID是1~499（不能登录系统）

​ 其他用户的UID是500~65535

创建用户的时候默认是创建在用户名同名的组中

用户的 账户信息，是放在 /etc/passwd中

arbor1 : 用户名

x ：密码占位符（以前是加密显示，因为不安全就显示x）

1000 ： UID

1000 ： GID（所属组）

/home/arbor1 : 用户的家目录

bin/bash : 这个用户所用的shell 信息

  • 密码放在 /etc/shadow 中

有效与初始化用户组

groups

groups userName

查看userName的所属组

当然也是可以让一个用户同时属于多个组

gpasswd -a arbor1 tom

将 arbor1 添加到tom组

arbor1 : arbor1 tom

​ 主组 副主组

newgrp

Linux newgrp 命令用于登入另一个群组。

我们是可以在图形化界面中添加用户的信息的

按照下面的图片就能找到了

特殊的sbin/nologin

使用useradd添加用户信息

选项 用途
-c 对应的就是一个类似简介的东西
-s 设置登陆的shell
-d 设置家目录
-g 设置谁是用户的主组 不创建自己用户的同名组
-G 创建自己同名的同名组，同时设置 后面的组名为副主组
-u 设置 UID

使用usermod更改用户信息

usermod可用来修改用户帐号的各项设定。

选项 描述
-c <备注> 　修改用户帐号的备注文字。
-d <登入目录> 　修改用户登入时的目录。
-e <有效期限> 　修改帐号的有效期限。
-f <缓冲天数> 　修改在密码过期后多少天即关闭该帐号
-g <群组> 　修改用户所属的群组。
-G <群组> 　修改用户所属的附加群组。
-l <帐号名称> 　修改用户帐号名称。
-L 锁定用户密码，使密码无效。
-s <shell>　修改用户登入后所使用的shell。
-u <uid> 　修改用户ID。
-U 解除密码锁定。

passwd命令

Linux passwd命令用来更改使用者的密码

选项 描述
-d 删除密码
-f 强迫用户下次登录时必须修改口令
-w 口令要到期提前警告的天数
-k 更新只能发送在过期之后
-l 停止账号使用
-S 显示密码信息
-u 启用已被停止的账户
-x 指定口令最长存活期
-g 修改群组密码
-i 口令过期后多少天停用账户
-n 指定口令最短存活期

一种 修改密码的方式

echo userPasswd | passwd –stdin userName

给用户账号上锁

passwd -l userName

usermod -L userName

两个命令效果是一样的

给用户解锁

passwd -u userName u == unlock

usermod -U userName

两个命令一样的效果

userdel命令

userdel -r userName

chage 命令

chage命令 是用来修改帐号和密码的有效期限。

  • chage -d

  • chage -E

  • chage -m

    我看视频理解的就是

    你修改问密码之后，要最少使用的天数

    防止你改完一个新的密码 第二天又改回旧的密码

  • chage -M

    理解：

    修改玩密码之后，做长不超过

    设置的最大天数就要修改一次密码

  • chage -W

    设置在密码过期前几天 提醒用户

  • chage -I

理解：

可以作为缓冲期 如

chage -I 3 bob

就是说bob 用他的这个bob账号 20天了还没有修改密码

那么在他登录的 21 22 23 天的时候

输入好了以前的密码，就会要求他更改新的密码

过了23天 他就不能在登录了
