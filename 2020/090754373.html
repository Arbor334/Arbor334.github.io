磁盘与文件系统管理

linux
发布日期:   2020-09-07
文章字数:   2.6k
阅读次数:  

硬盘与文件系统管理

block

（真正存储文件的地方）

默认是 一个小block 4k大小

如果一个block中存放数据了（存放了1k数据）

但是没有盛满（还有3k） 那么这block也是不可以再用的

inode

每个inode 是128字节 也就是说

记录一个文件占用一个inode 消耗128B

里面存的是block的相关属性 所有者 所属组 权限 创建时间等

系统查找文件时 先去inode中找 找到后在直接进入block拿数据

当我们分配block比较小 ，一个文件很大 ，占用很多block时，inode也会记录下来 这个文件占用了那几个block

super block

记录的是inode block的信息

super block

记录此文件系统的整体信息，包括 inode/block 的总量，使用量，剩余量 以及文件系统的格式与相关信息等(来自鸟哥linux私房菜）

文件系统描述信息

记录的是每个block group 从哪个块开始到哪个块结束

块位图

后面的block中那些块没有被用到 当我们添加新文件的时候，他就会查找块位图

inode位图

记录的是inode 信息 ，

inode表

每写入一个文件就 增加一条文件的信息 每个inode信息 128B 而每指向一个block 需要4B 所以最多只能记录32个block 一个block4KB也就意味着 一个inode信息 最多能记录128K的文件

命令

dumpe2fs [选项] 设备的名字（root用户下）

例 dumpe2fs /dev/hadc2

注意：centos 7下用 xfs_info +设备名字 centos 7用的是 xfs文件系统而不是 ext2文件系统

ls -i +文件名 显示的是这个文件名的inode编号

ls -i +目录名 显示目录名的inode编号

但是这个inode里面的记录的block存储的不是真实的数据

而是这个文件夹下面的其他条目的inode数据

在通过这些inode信息 去找其他条目下的文件

直到最后指向的inode存储的是真实数据block信息

filefrag -v +文件名 显示的是文件分布在哪些block里

日志

向block中写数据的时候

首先查找的是 inode表

看看里面那些inode没有被使用过 然后把数据的信息写进inode表中

然后去查找block位图，看看哪些block没有被使用过 然后就可以直接写进去

写进去之后，block位图和inode位图都要更改，super block也要更新

这个时候断电，block已经存储了数据，但是block位图或者super block还没有更新，信息不对等。

这时候用日志去修复就很方便了，没必要把整个磁盘都都修复了，大大节省了时间。

ext2没有日志 ext3有日志

journal inode 就是日志

虚拟文件系统

VFS

linux支持很多的文件系统 ，我们写东西存储的话，我们还要考虑是什么文件系统。有了VFS以后就不需要考虑了，VFS（虚拟文件系统）会自己会去调用文件系统。

显示挂载的设备

df

df 这个命令是调出目前挂载的的设备

df -T 显示出文件系统的类型

df -h 对人类友好的方式显示

查看文件夹的大小

du

du +文件夹名 查看文件夹大小（包括它所包含的文件）

du -s +文件夹名 只查看文件夹的大小

链接文件

软链接

（ln -s 源文件名 目的文件名）

相当于windows中的快捷方式

#   touch xx   创建名为xx的文件
#  ln -s xx yy  创建名为yy的 xx的软链接（快捷方式）

硬链接

（ln 源文件名 目的文件名）

硬链接和复制的区别是

硬链接创建的时候 在inode表中增加了一个inode 指向的是源文件所在的block 而复制文件 是新建一个inode 然后开辟新的block

#  ll
# -rwxrwxrwx 数字  这个数字就表示有多少个inode记录的是同一块的block（也可以说有几个名字）

硬链接不能跨分区

查看分区

->fdisk -l

显示的是系统中所有的硬盘和分区

fdisk -l /dev/sda1 后面加上一个硬盘号就能查看一个硬盘的

创建分区

-> fdisk +硬盘名（不是分区名）

一个硬盘最多只能划分4个主分区（主分区和逻辑分区都属于主分区），逻辑分区可以创建多个

当我们划分 分区 的时候是以柱面的形式划分的，也就是说柱面是分区划分的最小单位。我们是对硬盘进行分区，而不是对分区进行分区

（关于硬盘的基本知识链接）

创建主分区

①我们首先要进入这个硬盘，通过 fdisk /dev/+硬盘（不是分区）

②按m可以显示需要进行的操作

③ 按p可以查看硬盘的分区

④ 按n创建新的分区

->（分为主分区 p 和扩展分区 e）

->默认是创建主分区 p

->按回车 是默认起始扇区

->在输入要分配的大小 我这里演示是+100MB

->分配好了以后我们可以看到分区的类型是Linux ID是83

按 L 查看都有哪些分区的类型

按 T 改变分区类型 我们看到 这里Id 已经改变为了8e

-> 按D 删除分区 我们看到这里的 sda3已经没有了

创建扩展分区

(``扩展分区不可直接使用 要创建很多的逻辑分区`)

逻辑分区的盘符 默认是从5开始的

创建交换分区

像是windows下的虚拟内存

查看系统下的交换分区

cat /proc/swaps

第一种方法

划分一个分区出来

关闭扩展分区就是 swapoff /dev/sda6(分区名)

想要开机的时候还有这个交换分区 那么还是要写入到 /etc/fstab中

第二种方法

在某个目录里面创建一个文件夹，用dd命令创建存储数据的空间

  • /dev/zero
    “零”设备，可以无限的提供空字符（0x00，ASCII代码NUL）。常用来生成一个特定大小的文件。
  • if=文件名：输入文件名，默认为标准输入。即指定源文件。
  • of=文件名：输出文件名，默认为标准输出。即指定目的文件。
  • bs后面的 1如果没有M 单位是字节 而不是M

我们要把file占用当前分区的100M空间在作为swap

并不是不是file 这个文件作为swap

保存并退出

输入``w是保存并退出 q是不保存直接退出`

在输入命令 partprobe +硬盘名 是更新这个硬盘的数据表

如果 partprobe 后面不加硬盘名的话 更新的是电脑的所有硬盘的数据表

格式化后使用

刚分出来的 /dev/sda5 还不能使用 因为还没有格式化

只有格式化以后的block 才能够使用

命令：``mkfs -t 文件系统的类型 +硬盘名`

mkfs -t ext3 /dev/sda5 和 mkfs.ext3 /dev/sda5 作用是一样的

这就将 sda5格式化为ext3 文件系统 默认的block大小是 1024字节

mkfs -t ext3 -b 4096 /dev/sda5 ``-b指明每个block大小是多大`

挂载分区

mount 分区名 目录名（这个目录不为空的话，其他的数据就使用不了）

这就相当于 我在windows下创建了一个aa文件夹，我把某一个分区A装入到aa文件夹中来 ，以后我访问aa 就是访问分区A

下面的意思是 我以后访问 /mnt 就是进入了sda5这个逻辑分区

也可以使用 mount LABLE=yy /mnt 挂载到 /mnt中

LABLE 可以是分区的卷标

卸载挂载

umount 目录名

这个目录名和这个分区挂载时候的目录名要一样

磁盘检验

fsck +分区名

fsck -f +分区名 -f 强制检查

``-C 显示检查的过程`

检查坏道

badblocks +分区名 -sv 显示出进度和结果

检查这个分区中损坏的区块

磁盘的挂载和卸载

mount 外部设备 目录 这样只是临时挂载的 不是永久的

把这个设备 挂载到目录中

用``mount命令`也可以直接查看挂载的设备

在挂载的时候也可以使用权限

mount -o remount,ro 目录名

-o是有权限 remount 重新挂载 ro是read-only 这个命令运行以后

此设备只能读 不能写

rw 可读可写

noexec 没有可执行权限 就是rwx 中的x

exec 有可执行权限

开机自动挂载

如果要开机自动挂载挂载的话

vim /etc/fstab 进入这个目录中 去写东西

第一个 /dev/sda5  是要挂载的设备名
第二个  /mnt       是挂载点的名字
第三个  ext3      是文件系统
第四个  defaults   是默认的权限
第五个  0          是否要进行备份    0是不做 1是每天都做
第六个  0         是否用fsck进行自检  0是不检查 1是检查

umount 外部设备

umount 挂载点

挂载的会后后面的两个都要写

卸载的时候只需要用两个命令中的一个就可以了

在卸载的时候可能会提示 不能卸载正在运行

这个时候我们可以用

fuser -mv 挂载点 查看是哪个进程在用挂载点

挂载特殊设备

mount -o loop xx.iso /opt

以后详细的写

  • loop 写在 defaults 后面

查看磁盘参数

tune2fs -l 分区名

====

挂载次数超过最大挂载次数的时候 系统开始会强制自检这个分区

修改卷标名字

filesystem volume name

tune2fs -L xx /dev/sda5 就是将这个分区的 filesystem volume name 修改为xx

e2label /dev/sda5 yy 将 /dev/sda5的卷标修改为yy

查看卷标名字

e2label /dev/sda5

修改最大的挂载次数

Maximum Mount mount

tune2fs -c 100 /dev/sda5 将这个分区的Maximum Mount mount 修改为100

修改当前已经挂载的次数

Mount mount

tune2fs -C 101 /dev/sda5 将分区的Mount mount 修改为101

修改保留

Reserved block count 保留的空间 一般是5% 因为当你写满了数据，以后修改或者查找的时候就没有了地方下脚。

tune2fs -m 10 /dev/sda5 将10%改为保留的block
